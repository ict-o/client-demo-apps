# 案件進捗モニタリングシステム デモ

受注した案件が完了までに通る**確認事項（チェック項目）の進捗を、管理者が遠隔でリアルタイムに監視する**Web システムの提案用デモです。

営業・開発の担当者が各案件の確認事項（契約締結・入金確認・要件確定・中間報告・納品検収・請求など）を進めていく様子を、
管理者が離れた場所から一覧で把握します。**期限を超過した項目は「遅延」として自動検知**し、
担当者が**「不備」を報告すると管理者へ即時に通知（アラート）**が飛ぶ仕組みで、
案件が順調に進んでいるか・どこで止まっているかを可視化します。

- **性格: 業務システム型**（提案先の社内業務を置き換える／補助するシステム。システム名・UI文言・画面はすべて日本語）
- **種別: Viteビルド型**（React 19 + TypeScript + Vite。ビルドで静的ファイルを生成し GitHub Pages で閲覧）
- **公開URL**: https://ict-o.github.io/client-demo-apps/case-monitoring-demo/

## 想定ユースケース

- 案件を受注すると、完了までの**確認事項が自動でチェックリスト化**され、担当者・期限が割り当てられる
- 各担当者が確認事項を「着手 → 完了」で進めると、案件の**進捗率が自動で再計算**される
- 期限を過ぎた確認事項は**「遅延」として検知**され、管理者ダッシュボードとアラートに表示される
- 担当者が確認事項に問題を見つけたら**「不備を報告」**、その場で管理者へ通知が送信される
- 管理者はアラートを**確認済／対応済**に更新し、対応状況を管理する

## 主な画面・機能

| 画面 | 内容 |
|------|------|
| 監視ダッシュボード | KPI（監視中／遅延／要確認／未対応アラート）、リアルタイム通知フィード、対応が必要な案件のウォッチリスト、受注金額サマリ |
| 案件一覧 | 案件番号・案件名・顧客・担当・進捗率・状態・期限・受注金額をテーブル表示。検索＋状態／担当者での絞り込み |
| 案件詳細 | 案件情報、確認事項ごとの進捗ステップ（着手／完了／不備報告）、進捗率、活動履歴タイムライン |
| 新規案件登録 | 案件名・顧客・担当・金額・期限を入力して登録。標準の6確認事項が期限つきで自動設定 |
| アラート | 遅延・不備・期限接近の通知一覧。未対応／確認済／対応済で絞り込み、対応状況を更新 |

## 実演フロー（この順で最後まで操作できます）

1. **監視ダッシュボード**を開く（遅延中・要確認の案件、リアルタイム通知が並んだ状態）
2. 通知フィードの「遅延」通知をクリック → 該当案件の詳細へ遷移
3. **案件詳細**で確認事項のステップを確認。遅延している項目を「完了にする」→ 進捗率が上がり、関連アラートが自動で解消される
4. 別の確認事項で「不備を報告」→ 内容を入力して送信 → **管理者へ通知**（トースト表示、アラートが1件増える）
5. ヘッダーの「アラート」→ いま発報した不備通知を確認し、「確認済にする」「対応済にする」で対応状況を更新
6. 「案件一覧」→ 検索・状態／担当者で絞り込み → 「新規案件登録」で案件を追加 → 一覧・詳細に反映され監視が開始される

> 入力・操作した内容は画面上の状態に即時反映されます。ブラウザを再読み込みすると初期のサンプルデータに戻ります（永続化はしていません）。

## 使用技術

- React 19 + TypeScript
- Vite（`base` をサブパスに設定し、GitHub Pages でそのまま閲覧できる静的ファイルを生成）
- React Router v7（`HashRouter`）
- ランタイム依存の外部サービス・バックエンドなし（フロントエンドのみで完結）

## ビルド方法

```bash
npm install
npm run build
```

- ビルド成果物（デモ直下の `index.html` と `assets/`）が公開対象です。`src/` は開発用ソースです。
- `main` へマージされると GitHub Pages に公開されます。

## データについて

- 表示されている**企業名・担当者名・案件名・金額・日付はすべて架空のサンプル**です。
- 実在の顧客・案件・個人情報や、認証情報・実データは一切含みません。
- 運用会社（当社）名「アオイ商事株式会社」も架空です。
