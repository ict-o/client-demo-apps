# 作業予定管理システム（ビルメンテナンス業向け）デモ

ビルメンテナンス業（病院・ホテル・商業施設・オフィスビル等の清掃／設備／衛生管理）における
**作業の日程調整〜作業報告までを一元管理する Web システム**の提案用デモです。

顧客・当社（元請け）・協力業者の三者が同じシステム上で、
候補日の提示、OK／NG回答、再調整、日程確定、作業開始、写真登録、報告書の作成・提出までを
完結できることを示します。電話・FAX・メールによるアナログな日程調整からの脱却を狙った提案イメージです。

- **性格: 業務システム型**（提案先の社内業務を置き換える／補助するシステム。UI 文言はすべて日本語）
- **公開URL**: https://ict-o.github.io/client-demo-apps/maintenance-scheduling-demo/

## 想定ユースケース

- 定期清掃・設備点検・保守作業などの**作業予定を登録**し、顧客へ**候補日を提示**する
- 顧客がブラウザ上で各候補日に **OK／NG を回答**（NG時は理由を添付）
- NGの場合は当社が**候補日を再提示**して再調整
- OKの候補日で**日程を確定**（確定作業はカレンダーに反映）
- 協力業者が**作業を開始**し、チェック項目の消化・**現場写真の登録**・**作業報告書の作成／提出**を実施
- すべてのやり取りを**履歴（誰が・いつ・何を）**として自動記録し、担当者依存・確認漏れ・履歴管理のしづらさを解消

## 主な画面・機能

| 画面 | 内容 |
|------|------|
| 作業一覧 | KPI（提示中／確定／作業中／完了）・検索・ステータス／作業種別の絞り込み・並び替え。役割に応じて閲覧範囲を制御 |
| 作業詳細 | 概要／日程調整／作業報告／履歴 の4タブ。現在の状況に応じた「次のアクション」を提示 |
| 新規作業の登録 | 顧客・施設・作業種別・協力業者・担当・候補日を入力して登録（当社のみ） |
| 作業カレンダー | 日程確定した作業を月別に表示。前月／翌月の切替、当日ハイライト |

## 三者の視点（表示視点の切替）

画面右上の「表示視点」で **当社（元請け）／顧客／協力業者** を切り替えると、
同じシステムを各立場から確認できます（デモ用のアカウント固定切替）。

- **当社**: 全案件の管理、新規登録、候補日提示、日程確定
- **顧客**（さくら総合病院）: 自社宛の作業のみ表示。候補日への OK／NG 回答、報告書の閲覧
- **協力業者**（みどり環境サービス）: 担当作業のみ表示。作業開始、写真登録、報告書の作成／提出

## 実演フロー（この順で最後まで操作できます）

1. **当社**視点で「作業一覧」を確認（進行中の案件が多様なステータスで並ぶ）
2. 「新規作業を登録」→ 顧客・作業種別・施設・協力業者・候補日を入力して登録 → 一覧に「候補日提示中」で出現
3. 表示視点を**顧客**（さくら総合病院）に切替 → 対象案件の「日程調整」タブで候補日に **OK／NG** を回答
4. 表示視点を**当社**に戻す → OK の候補日で「この日程で確定」（NGのみなら「候補日を再提示」で再調整）
5. 「作業カレンダー」で確定作業が反映されていることを確認
6. 表示視点を**協力業者**（みどり環境サービス）に切替 → 確定案件で「作業を開始」→ チェック項目を消化 → **作業前／作業後の写真を追加** → 報告書を入力 → 「報告書を提出」
7. 表示視点を**顧客**に切替 → 提出済みの報告書を閲覧（業務完了）
8. 「履歴」タブで全操作の記録（誰が・いつ・何を）を確認

> 入力・操作した内容は画面上の状態に即時反映されます。ブラウザを再読み込みすると初期のサンプルデータに戻ります（永続化はしていません）。

## 使用技術

- React 19 + TypeScript
- Vite（`base` をサブパスに設定し、GitHub Pages でそのまま閲覧できる静的ファイルを生成）
- React Router v7（`HashRouter`）
- ランタイム依存の外部サービス・バックエンドなし（フロントエンドのみで完結）

## ビルド方法

```bash
npm install
npm run build
```

- ビルド成果物（デモ直下の `index.html` と `assets/`）が公開対象です。`src/` は開発用ソースです。
- `main` へマージされると GitHub Pages に公開されます。

## データについて

- 表示されている**企業名・施設名・担当者名・作業内容・日程はすべて架空のサンプル**です。
- 実在の顧客・案件・個人情報や、認証情報・実データは一切含みません。
- 運用会社（当社）名「セントラルビルメンテナンス株式会社」も架空です。
